# Linux System Config (Arch + HyDE)

## Purpose
Single source of truth for system-level configurations on the Arch + HyDE + Hyprland setup. Covers things outside of `~/dotfiles/` (kernel, bootloader, systemd, /etc) that would be lost on reinstall.

## Hardware
- **Model**: Razer Blade 15 (Mid 2021)
- **BIOS**: AMI Aptio v2.21.1278 (does NOT expose Setup Mode / Custom Mode for Secure Boot)
- **GPU**: Hybrid Intel iGPU (i915) + Nvidia RTX 3060 (nvidia-open-dkms)
- **Filesystem**: btrfs

## Bootloader & Kernel

### `/etc/mkinitcpio.conf` — required modules
Hybrid Intel+Nvidia needs both drivers loaded in the initramfs, otherwise boot stalls ~1 min waiting for the iGPU.

```
MODULES=(i915 nvidia nvidia_modeset nvidia_uvm nvidia_drm btrfs)
```

Regenerate after any change:
```bash
sudo mkinitcpio -P
```

### Secure Boot — DISABLED
BIOS limitation: only exposes Enable/Disable, no Setup Mode or Key Management. Cannot enroll sbctl keys. **Left disabled** as accepted risk.

For real data protection add LUKS full-disk encryption (not currently set up, ~2-3h migration).

## Display Manager

### SDDM autologin
HyDE ships an empty `[Autologin]` block in `/etc/sddm.conf.d/`. Override with `/etc/sddm.conf.d/99-autologin.conf`:

```ini
[Autologin]
User=andre
Session=hyprland
Relogin=false
```

Enable unit:
```bash
sudo systemctl enable sddm
```

## GPU / Wayland

### `~/.bash_profile` — AQ_DRM_DEVICES
Hyprland uses Aquamarine (not wlroots) — needs explicit device order:

```bash
export AQ_DRM_DEVICES=/dev/dri/by-path/pci-0000:00:02.0-card:/dev/dri/by-path/pci-0000:01:00.0-card
```

Intel first (primary output, less power), Nvidia second. `WLR_DRM_DEVICES` is legacy, do NOT add.

## Hyprland / HyDE

### Updating HyDE
```bash
cd ~/HyDE
git pull
./Scripts/install.sh -rn    # restore configs, skip nvidia steps
```

`-r` syncs `~/.local/share/hyde/`, `~/.local/lib/hyde/`, and populates HyDE template files in `~/.config/hypr/` (only if missing — user edits preserved).

### Hyprland v0.53+ windowrule syntax migration
Hyprland 0.53+ broke the old windowrule syntax. Pattern:

| Old | New |
|---|---|
| `class:^(...)$` | `match:class ^(...)$` |
| `title:^(...)$` | `match:title ^(...)$` |
| `initialTitle:^(...)$` | `match:initial_title ^(...)$` |
| `windowrule = float, ...` | `windowrule = float true, ...` |
| `windowrule = idleinhibit fullscreen, ...` | `windowrule = idle_inhibit fullscreen true, ...` |
| `windowrule = keepaspectratio, ...` | `windowrule = keep_aspect_ratio true, ...` |
| `layerrule = ignorezero, ...` | `layerrule = ignore_alpha 0, ...` |

PiP now uses block syntax `windowrule { ... }`. Reference: `~/HyDE/Configs/.config/hypr/windowrules.conf`.

User's customized file lives in `~/dotfiles/hyprland/.config/hypr/windowrules.conf` (stowed).

### Waybar management
Waybar is launched via systemd transient unit `hyde-Hyprland-bar.service` managed by `~/.local/lib/hyde/waybar.py --watch`.

**DO NOT** `pkill waybar` — the watcher respawns it outside systemd → orphan process survives future theme switches (causes double-waybar bug).

Correct commands:
```bash
systemctl --user restart hyde-Hyprland-bar.service
systemctl --user status hyde-Hyprland-bar.service
```

If a double waybar appears, find the non-systemd PID (`systemctl --user status hyde-Hyprland-bar.service` shows the managed one) and `kill <orphan_pid>`.

### Wallpaper backend: awww (not swww)
Arch's `extra/swww` now resolves to `awww` — same project renamed by LGFae. HyDE v26+ ships `wallpaper.awww.sh` and calls `awww`/`awww-daemon` directly. Don't try to install `swww` separately.

Check daemon: `pgrep awww-daemon`
Query current: `awww query`
